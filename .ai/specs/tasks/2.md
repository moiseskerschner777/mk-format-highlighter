# Phase 2 — Inline add-comment input + Enter-to-stack

**Goal:** Selecting text in the editor surfaces an inline input; pressing Enter stacks a real comment via `MkCommentManager` and shows a gutter icon on that line.

**Flow:** Flow 1 — Build→Observe→Adjust

**References:** `design.md` → Component contracts (`AddCommentInlineAction`, `MkCommentGutterProvider`), Architecture decisions (inline input, not modal)

---

> **When a task is complete, mark it `[-]`.**

- [-] 2.1 Create `AddCommentInlineAction.kt` registered as an editor action (context menu entry "Add Comment" + keyboard shortcut), enabled only when there is a non-empty text selection.

  **Test:**
  - Do: run `./gradlew runIde`, open `sample.mk`, select any line of text, right-click
  - Expect: "Add Comment" appears in the context menu; right-clicking with no selection does not show it (or shows it disabled)

- [-] 2.2 Implement the inline input itself: on triggering the action, insert a small text field directly below the selected line (per the reviewed mockup), auto-focused.

  **Test:**
  - Do: select `mk-format-highlighter:` on line 2, trigger "Add Comment"
  - Expect: a text input box appears directly under line 2, cursor focused inside it, rest of the file unaffected

- [-] 2.3 Wire Enter inside the inline input to call `MkCommentManager.addComment(editor, request)` with the typed text, then dismiss the input.

  **Test:**
  - Do: with the inline input open from 2.2, type "rename this node to mk plugin" and press Enter
  - Expect: input closes, no error in the log

- [-] 2.4 Wire Escape inside the inline input to dismiss it without creating a comment.

  **Test:**
  - Do: trigger "Add Comment" on a line, type a few characters, press Escape
  - Expect: input closes, "Log mk Comments" (from Phase 1) shows no new comment added

- [-] 2.5 Create `MkCommentGutterProvider.kt` implementing `LineMarkerProvider`, rendering an icon on any line with ≥1 comment, registered in `plugin.xml`.

  **Test:**
  - Do: repeat the stack action from 2.3 on line 2, then look at the gutter
  - Expect: a small icon appears in the gutter next to line 2

- [-] 2.6 Stack a second comment on a different line and confirm both gutter icons persist independently.

  **Test:**
  - Do: select `color palette:` (line 31 in `sample.mk`), trigger "Add Comment", type a note, press Enter
  - Expect: gutter icons now show on both line 2 and line 31, each independently clickable-looking (click behavior comes in Phase 6)
